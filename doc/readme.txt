
TraceLogAnalyser 项目说明：

1、这是一个用于代码分析的工具，与 BTrace 配合使用，便于分析代码的调用热点；

2、通常，首先编写 BTrace 脚本对目标代码的调用进行捕捉，并将捕捉到的调用栈输出到日志文件，通过此工具进行分析和展示调用栈；

3、BTrace 脚本的日志输出逻辑需要符合以下要求：
		（1）每次捕捉到的调用栈日志片段需要输出一个独立的开始行和结束行，以便 TraceLogAnalyser 可以在一份日志文件中区分对目标代码的多次调用的调用栈；

4、TraceLogAnalyser 将从日志文件提取的多个调用栈日志片段合并分析，将全部的调用栈的执行路径合并为一个代码执行路径图，在分析结果页面中展示为代码执行路径树；

5、TraceLogAnalyser 的分析结果与 jvisualvm 的 cpu analyse 的输出类似，此工具的输出类似于将 jvisualvm 的 cpu analyse 的快照中的反向追踪的结果倒置为正向，
   使用正向视图更便于展示目标代码被调用的频度在调用栈高层的分布；
